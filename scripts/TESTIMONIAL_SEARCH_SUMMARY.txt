THE PROBLEM
================================================================================

ISSUE: Cross-category contamination in testimonial search
- Dog grooming testimonials appearing for dental prospects
- Plumbing testimonials showing up in restaurant searches
- System mixing categories and ignoring business-type boundaries
- No exclusion lists preventing wrong-category results

ROOT CAUSE:
- Generic keyword expansion (e.g., "salon" matched both hair + dog grooming)
- No category boundaries enforced
- Fallback chains mixing unrelated industries
- No exclusion mechanism

IMPACT: Prospects seeing irrelevant testimonials, reducing credibility

================================================================================
THE SOLUTION
================================================================================

NEW SYSTEM: Category-Specific Keyword Chains with Strict Boundaries

Architecture:
1. CATEGORY_FALLBACKS data structure (47 subcategories)
   - Primary keywords (most specific)
   - Fallback chains (tried in order until 3+ results)
   - Exclude lists (keywords that disqualify results)
   - Category exclusions (entire categories barred)

2. Keyword extraction from business names
   - "Mountain View Dental" → "dental"
   - "The Fluffy Ruff Dog Spa" → "dog grooming"
   - "Joe's Plumbing" → "plumbing"

3. Smart search with fallback chains
   - Try primary keyword first
   - If <3 results, try next fallback
   - Keep trying until 3+ results or end of chain
   - NEVER cross category boundaries

NEW FUNCTIONS
================================================================================

extract_business_keyword(business_name: str) -> str
  - Extract primary business type from name
  - Handles multi-word names, removes articles
  - Example: "The Fluffy Ruff Dog Spa" → "dog grooming"

get_category_fallback_keywords(category: str, subcategory: str) -> List[str]
  - Return complete keyword chain for a category
  - Example: "🏥 Health/Medical"/"Dentist" → 
    ["dentist", "dental", "orthodontics", "orthodontist", ...]

get_exclusion_keywords(category: str, subcategory: str) -> (List, List)
  - Return exclusion keywords and categories
  - Example: Dentist excludes ["grooming", "dog", "plumbing", ...]

search_testimonials_by_keyword(keyword: str, testimonials: List) -> List
  - Search testimonials by single keyword
  - Basic search building block

search_testimonials_by_category(category: str, subcategory: str, ...) -> List
  - Smart category-aware search with fallback chain
  - Enforces exclusions, respects boundaries
  - Core search function

get_testimonials_for_prospect(prospect: Dict, category: str, subcategory: str) -> List
  - Main function for getting prospect testimonials
  - NEW SIGNATURE (includes category + subcategory)
  - Returns up to 3 relevant testimonials
