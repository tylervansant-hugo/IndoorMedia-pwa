# 2026-04-27 — Session Notes

## PWA Deployment Fix (CRITICAL — remember this!)

### Root Cause Chain
1. **GitHub Pages serves from `gh-pages` branch, NOT `main`** — We were pushing builds to `main` and wondering why nothing changed. Must use `npx gh-pages -d dist` to deploy.
2. **Service worker v3 was cache-first** — Old SW cached everything, never checked network. Bumped to **v4-network-first** in `public/service-worker.js`. HTML/JSON now use network-first strategy.
3. **No cache-busting on data fetches** — Added `?t=' + Date.now()` to ALL `data/*.json` fetch calls across every component (18 Svelte files updated). GitHub Pages CDN caches aggressively; query params bypass this.
4. **JavaScript UTC date parsing bug** — `new Date('2026-04-01')` parses as UTC midnight → March 31st in Pacific time. Created `parseContractDate()` helper in Main.svelte that strips time, parses date-only at noon local time. Handles both `YYYY-MM-DD HH:MM` and `M/D/YYYY` formats.
5. **Contracts.json divergence** — The nightly scan wrote to local `data/contracts.json` but only pushed to `main`, not `gh-pages`. Merged 5 missing April contracts (El Ricon Taqueria, Novus Glass, Men's Barber, Ridgefield Travel Center, Green Bee's Landscaping).

### Deployment Workflow (REMEMBER)
```
npm run build          # Builds to dist/
npx gh-pages -d dist   # Deploys dist/ to gh-pages branch
git add -A && git commit && git push origin main  # Source code to main
```
**Both steps needed!** `git push` alone does NOT update the live site.
